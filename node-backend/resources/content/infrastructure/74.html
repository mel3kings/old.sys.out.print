Once you have kubernetes up and running, you might need to install dashboard. As it might be more difficult trying to check pods using command lines. 

How to Install

  1.  kubectl create -f https://raw.githubusercontent.com/kubernetes/dashboard/master/src/deploy/recommended/kubernetes-dashboard.yaml
  2.  kubectl proxy
You will be able to access the dashboard via:
http://<ip>:8001/api/v1/namespaces/kube-system/services/kubernetes-dashboard/proxy/#!/overview?namespace=default

Things you can do on the Dashboard:

  1. Let's start with the obvious, you can see the status of Deployments and your existings pods
  1. As per above, you can see there's a create button on the upper right, in here you can create docker based applications. 
    • This can be either pulled from a docker repo by just specifying the image name or
    • Uploading Docker yaml file or
    • Text input where you create your own Dockerfile through the dasbhboard (this might not be the best way to do it as you will be debugging your yml file through the dashboard)
  2. On the left hand side, you can check Pods, to see the status of each Pods and more details. 
  3. Another feature worth noting is how you can see individual logs for each pod (this through pods menu) 
  4. You can see CPU/Memory details as well through the dashboard

A more detailed explanation about the UI dashboard can be found here:
