File I/O in python is quite straightforward.
The command below will try to open an file that is located in the current working directory. (use pwd to print out where you notebook is running)
myFile = open('test.txt')

Common Error is FileNoteFoundError [Errno 2] - this will usually occur if you used a wrong file name and/or wrong path

Writing a File
with open('test.txt', mode='w+') as f:
    f.write("hello")
Breaking down the codes:
  • with open acts as a wrapper to the semicolon located at the end of the line
    • Basically it says open the file but once the codes has been executed close the file
    • mode depicts what kind of i/o you are performing, indicating a wrong mode will throw an error (ie: setting mode as read but performing write operations)
      • r - read
      • w - write
      • a - append
      • r+ - read and write
      • w+ - read and write
Reading a File:
with open('test.txt', mode='r') as f:
    print(f.read())
Similar to the break down above except we have print, to see what we are reading, and mode set to read

Doing it manually:
Below is the brute force way of reading a file, without using a with open:
readingFile = open('test.txt') // opens the file
print(readingFile.read()) // reads and prints out the line
print(readingFile.read()) // NOTE: this will return empty, because at this point the cursor has already been led to the end of the                           // file due to the first call to read
readingFile.seek(0) // this is how we set the cursor to the beginning of the file
print(readingFile.read()) // we can print again
readingFile.close() // we need to manually close the file as it is currently being accessed by Python
