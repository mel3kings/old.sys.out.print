Events are in another word triggers, that the application performs certain actions. 

NodeJS generally does not have Event objects, so Node just creates their objects to simulate events that the applications react to.

Node Event Emitter creates a pseudo event listener, whereas you can specify different set of actions given an event. This is a common concept in programming as well. 

  1. Below we have three events, TEST, SAVE, EXIT. 
  2. We have a Node library 'events'
  3. We create specific actions with 'on'
  4. So for example, on event SAVE do the following action and so on
  5. We invoke the event listener we do 'emit'

var Emitter = require('events');
var emtr = new Emitter();
var events = {
    TEST: 'test',
    SAVE: 'save', // this is the same concept above where we a dynamically invoking methods, this lessens the probability of a typo
    EXIT: 'exit'
};

emtr.on(events.SAVE, function(){ // same concept as above whereas it is actually just an array, but inside it there is an if switch to invoke only if events is SAVE
   console.log('Event saved');
});

emtr.on(events.exit, function(){
    console.log("Event exit");
});

emtr.on(events.TEST, function(){
    console.log('code testing');
});

emtr.emit(events.SAVE);
emtr.emit(events.TEST);
emtr.emit(events.EXIT);
