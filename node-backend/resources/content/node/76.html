In order to make Node more modular we make use of exports and require. In traditional programming, we wouldn't want to have all the files in one big file. 

Modules are a small block of functionality, that together performs an intended purpose.

exports (module.exports) - enables a function/object to be accessible to other js files
require - imports that function/object

For Example:
We have a service, that wants to invokes doing work and then perform a greeting, and we don't want all those methods/functions in one js file.

Greeter.js
var greetMe = function () {
    console.log('hello again!')
}
greetMe()

module.exports = greetMe;
module.exports - this line, we are exporting just the function itself.

Worker.js
function doWork(){
    console.log("doing work");
}

var data = "Some database data";
module.exports = {
    doWork: doWork,
    data : data
};
Worker on the other hand, we want to export more than just a function, we want to export some data AND a function, so the syntax is above.

Finally to glue it altogether:
Service.js
var worker = require('./Worker');
var greeter = require('./Greeter');

worker.doWork();
console.log("Now i have access to: " + worker.data);
greeter();

At this point Service, now has all the access to the methods/functions/variables from other classes. This enables us to make Node more modular.
