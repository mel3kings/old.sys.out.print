Given n what is the sum of all number up to n

This is a common problem in programming world, and we will discuss what are the common solutions that we use and how to optimize it. let's dive right in:

Now in order to add all the sums up to 100, let us first visualize whats going on

Now it is easy to for the easy solution, and just iterate over it right? and using simple iteration we would have something like: 

1 + 2 + 3 ... + 100
      public static int manuallyAdding(int n) {
        int sum = 0;
        for (int i = 0; i < n; i++) {
            sum += i;
        }
        return sum;
    }

This is well and good, if you only have a few data set. like up to 10 only, but what happens when you have an extraordinarily big number? like 99999999? How is iterating going to affect your memory for the application. there will be a performance hit for sure. Here is a more optimized way to do it

First carefully look at all the number, notice that if you add the ends of the numbers going to the center they are all the same for example:

    1 + 100 = 101
    2 + 99 = 101
    3 + 98 = 101
    ...
    50 + 51 = 101

they are all equal to n + 100, remember our n was 100, and all the sums of the ends are all equal to n + 1 = 101 and notice that if you get n / 2 pairs of 101's. Drawing from this logic we can say that the summation of numbers are: 

    (n/2)(n + 1)
    50 * (100 + 1)
    50 * 101 = 5050

So we can always use this formula rather than iterating over the values, which is much faster as well.
