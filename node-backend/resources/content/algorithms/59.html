In the computer science world, there are terminologies that we use to better explain concepts.
Algorithm is just step by step instructions on what you want your machine to do. Any block of codes can be considered an algorithm. 
Efficiency of Algorithm is dependent on how well it performs given that we have more input to it.

Growth Rates
  • Linear algorithm - algo scales with the number directly the same number of input is considered linear.
    • 10,000 input means 10,000 times the algo runs.
  • Constant Algorithm - same run time regardless of input

  • Quadratic Algorithm- n^2 size of the input squared
    • Quadratic simple means squared 
    • For example you have 10 values in an array, a nested loop would execute that 100 times. 
Asymptotic Notation/Big O Notation
Do not be afraid of this big word, it is just simple a term or way of distinguishing algorithms in a universal way and not just in english. Cause it would be hard to explain to people who do not speak English to talk about Linear, Constant, etc. Hence Asymptotic Notations were introduced.  Also known as Big O Notation.
  • Linear - O (n)
  • Constant - O(1) - we dont care about the input, usually in Maps
  • Quadratic - O(n^2)
  • Cubic - O(n^3) - three nested loops
  • Exponential - O(n^n) - those algorithms that really explode! e.g: 10 inputs resulting in a million operations
