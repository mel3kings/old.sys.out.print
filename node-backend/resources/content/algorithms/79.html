Protocol - set of rules that two sides agree on to use when communicating
  • Both client and server agree to use that particular set of rules to establish communication
  • For example, your web browser uses HTTP protocol (Hyper Text Transfer Protocol)


IP - Stands for Internet Protocol
  • IP sequence of numbers that identifys a web server and client
  • When that connection is established, socket is open to enable data flow
  • Information sent are sent formatted as well through different protocols:
    • HTTP - HyperText Transfer Protocol, for web uses
    • SMTP - Simple Mail Transfer Protocol, for mail uses
    • FTP - File Transfer Protocol, for file transfers

TCP - Stands for Transmission Control Protocol
  • Although data have their own protocols (http, smtp, ftp), TCP is the one in charge of breaking down the data into small packets that are sent through the web, through a Socket.
  • Similar to the concept of Streams and Buffers



TCP and IP usually goes hand and hand hence they are always referred to as TCP/IP
