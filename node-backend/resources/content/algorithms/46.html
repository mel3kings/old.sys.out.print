Big O Notation - programmers may often have hear this phrase but don't grasp what it really means. It sounds daunting at first, and quite complex to understand. But really it's just a way of saying how fast a function/method is. This is usually a measurement of how fast the method can be executed, and can be a measure if your code is not optimal.

Let's dive in, here are four basic big O notation, there are four basic notations (there are more but wont be covered here)

Most basic is O(1) this denotes that a method will return with the same speed regardless of the input. an example is below where you check if a string is empty or not. whatever String we put it will always return speed.

O(1)

private static boolean isNull(String empty) {
 return empty == null;
}

Next is O(n) which in contrast to O(1) depends on the size of the input. As the input grows the longer it takes to execute.

O(n)

private static void isNull(List<String> empty) {
 for (String data : empty) {
 System.out.println('processing ' + data);
 }
}

Next we have O(N2) which is similar to the above, 
but with nested loops, specifying that the operation time can easily square the size of the data set
if you perform nested loops which is not really ideal. (one benefits of using streams is to avoid nested loops)

O(N2)

private static void isNull(List<String> empty) {
 for (String data : empty) {
 System.out.println('processing ' + data);
 for (String nested : empty) {
 System.out.println('processing again ' + nested);
 }
 }
}

Finally we have O(2N) represents an algorithm whose performance is directly proportional to the square of the size of the input data set, this usually occurs in recursive data sets, and execution time can easily rise exponentially.

O(2N)

private static int Fibonacci(int number) {
 if (number <= 1) return number;
 return Fibonacci(number - 2) + Fibonacci(number - 1);
}
