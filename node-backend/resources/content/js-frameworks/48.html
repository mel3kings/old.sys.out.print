ES6 Basic Syntax

As most of you are already aware we have js-frameworks already that further the Javascript as a language. At the core of these frameworks sits ES6, it gives additional flexibility/functionality to the plain vanilla JS Here are some basic syntax to work as a refresher for you if you are just starting out playing around these frameworks

New Variable Types


Previously in Javascript you would declare variables using the following syntax: 
    var a = "hello";
JS did not have any notion of what kind of variable is this, var is one size fits all. With ES6, comes to variation of var initialisation 
    let a = "hello";
    const b = "world";
  • let - is a temporary variable, that is expected to change this is typically used when you that the var is constantly changing
  • const - in contrast, const is a final variable, and will throw syntax error when you try to reassign, this allows for perfomance improvements since the engine will know that there will be no reallocation of this variable and does not need to keep track of its changes.
A typical use of const in React would be, when you want to specify a constant that could be access globally by your application, for example below: 
    export const RENDER_TECH_PAGE = 'render_tech';
on the other files, you would just import the variable: 
    import {RENDER_TECH_PAGE} from "../actions/action_render_page";

Concatenation using ... and ``

In ES6 we can concatenate using the syntax ...
Consider the following example: 
    let a =[1, 2 ,3];
    let b = [6, ...a ,10];
    console.log(b);
This will output: 6 1 2 3 10
where as previously you had to do manual concatenation of variables, or method calls using .concat() On the other hand, we can use the same topic, we can do deconstruction as well consider the following: 
    let people =['Greg', 'Mel'];
    let [person1, person2] = people;
    console.log(person1, person2);
The essentially maps your array constants to the new variables.
Pitfall if you already have a different assignment to certain variables you can use this syntax to reassign it, consider the following below: 
    let person1 = 'Mike';
    let person2 = 'Agnes';
    let people =['Greg', 'Mel'];
    ([person1, person2] = people);
    console.log(person1, person2);

String Manipulation

Previously in Vanilla JS to add String together you would do the below: 
    var a = 'Hello';
    var b = 'World';
    var c = a + ' ' + b;
    console.log(c);
New of doing it is to use `` 
    let luke = 'blue';
    let vader = 'red';
    let message = `Luke first uses a ${luke}-colored lightsaber';
    console.log(message);
`` - tells the engine to translate the variables, this is essentially usefully when dealing with alot of string manipulation, instead of bombarding it with + to concat Strings
