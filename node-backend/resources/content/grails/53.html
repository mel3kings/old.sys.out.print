Grails

is web application framework that hopes to eliminates the complexity when building/setting up a new application. Think about how you setup all the things from scratch, configuring tomcat server, creating object relational mapping, setting up spring boot etc.
  • Built in key features
    • Spring/Hibernate built in
    • GORM - hibernate mapping
    • GSP - view technology
    • Controllers are done 
    • Embedded Tomcat server - for hot deploy (no more restarts!)
    • instant i18n support
    • Transactional service

Script Oriented Framework

Once grails is installed, there are commands you need to remember, they are usually used when automatically creating application/controllers.
For example:
You can create rest app by running this command, this will prepare all the necessary folders/structures you need:
$ grails create-app myapp --profile=rest-api
   
You can create a rest controller by running this command
grails> create-controller hello
in turn it will create the following for you:
  1. HelloController.groovy
  2. HelloControllerSpec.groovy (test case)
  3. create a hello folder under views 
  4. Depending on which grails version it might update your application.yml as well

Running the below command will run that rest application:
grails> run-app

Now that was quick, to create a web rest service, it was essentially just running two commands! 
Other commands include, there are alot more create-scripts that grails support, head over to their documentation with appropriate version to check them out.
  • test-app - runs all the test under test folders
  • war - packages application

Folder Structure convention:

  • grails-app - top level directory for Groovy sources

    • conf - Configuration sources

    • controllers - Web controllers - The C in MVC.

    • domain - The application domain.

    • i18n - Support for internationalization (i18n).

    • services - The service layer.

    • taglib - Tag libraries.

    • utils - Grails specific utilities.

    • views - Groovy Server Pages or JSON Views - The V in MVC.

  • scripts - Code generation scripts.

  • src/main/groovy - Supporting sources

  • src/test/groovy - Unit and integration tests.
Some files to note that are automatically created for you (depending on grails version): 
  • BuildConfig.groovy/build.gradle
  • Config.groovy/application.groovy
  • UrlMappings.groovy
  • BootStrap.groovy

grails url-mappings-report - shows all existing urls
